# Network Traffic Application Identification

Machine learning pipeline for classifying IP network traffic flows by application (e.g., YouTube, Microsoft OneDrive, Skype, Dropbox, Google) using flow-level features such as packet size statistics, inter-arrival times, and flow duration.

Developed as part of the **Network Data Analysis Laboratory** course at Politecnico di Milano (Prof. Francesco Musumeci, DEIB).

---

## Motivation

Network operators apply different priorities to different traffic types — congestion control, resource allocation, QoS, security. Layer-4 port numbers alone are no longer sufficient to identify modern applications, so flow-level statistical features combined with machine learning offer a practical alternative.

This project trains and compares classifiers that take a feature vector describing a traffic flow and predict the originating application.

---

## Dataset

- Source: [IP Network Traffic Flows Labeled with 87 Apps](https://www.kaggle.com/datasets/jsrojas/ip-network-traffic-flows-labeled-with-87-apps) (Kaggle)
- 22 applications, 87 features per flow
- Features include: source/destination IP and port, protocol, flow duration, forward/backward packet length statistics (mean/min/max), inter-arrival times, flow bytes/s, flow packets/s, etc.

The raw CSV is not committed to this repo due to size — see [Setup](#setup) for download instructions.

## Pipeline

1. **Dataset creation** — extract flows for the chosen applications and save a per-pair feature CSV.
2. **Data visualization** — plot per-feature distributions per class to inspect separability.
3. **Dataset loading** — build `(X, y)` arrays, perform an 80/20 stratified train/test split.
4. **Hyperparameter optimization & training** — 5-fold cross-validation on:
   - Logistic Regression (with and without feature standardization)
   - XGBoost (grid over `n_estimators`, `max_depth`, `subsample`)
5. **Testing & performance evaluation** — confusion matrices, accuracy, precision, recall, F1 (weighted).
6. **PCA visualization** — 2-component projection to motivate model performance differences.

---

## Results — Binary Classification (MS_ONE_DRIVE vs YOUTUBE)

| Model | Accuracy | Precision | Recall | F1 |
|---|---|---|---|---|
| Logistic Regression (standardized) | 0.723 | 0.738 | 0.723 | 0.718 |
| XGBoost | 0.967 | 0.967 | 0.967 | 0.967 |

XGBoost substantially outperforms logistic regression. PCA visualization shows the two classes are not linearly separable — the boundary between OneDrive and YouTube flows is non-linear in the feature space, which suits a tree-ensemble model and explains the gap.

## Multi-class Extension (Skype / Dropbox / Google)

The notebook extends the same pipeline to a 3-class problem, adapting:

- `load_dataset()` to assign three labels (Skype = 0, Dropbox = 1, Google = 2)
- Logistic regression with `multi_class='multinomial'`
- 3×3 confusion matrices and weighted-average metrics

## Setup

```bash
# clone
git clone https://github.com/amanzargari/Network-Traffic-App-Identification.git
cd Network-Traffic-App-Identification

# environment
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# data — download from Kaggle and place in data/
# https://www.kaggle.com/datasets/jsrojas/ip-network-traffic-flows-labeled-with-87-apps
```

Then open `notebooks/app_identification.ipynb` and run top-to-bottom.
